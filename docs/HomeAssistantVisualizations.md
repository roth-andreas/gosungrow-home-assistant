# Home Assistant Visualizations

This repository can drive two different visualization styles in Home Assistant.

## Recommended Today: Official Energy Cards

Home Assistant now has built-in energy cards that are a better long-term solution than the older custom Lovelace setup.

Useful official cards for a Sungrow system are:

- `power-sankey` for live power flow
- `energy-distribution` for energy flow over time
- `power-sources-graph` for live power history
- `energy-usage-graph` for home consumption
- `energy-solar-graph` for solar production
- `energy-sources-table` for grid / solar / battery totals

Official docs:

- https://www.home-assistant.io/dashboards/energy/
- https://www.home-assistant.io/docs/energy/faq/

### What To Configure In Home Assistant Energy

Go to `Settings > Dashboards > Energy` and map your Sungrow sensors there.

Suggested mapping for a Sungrow ESS device:

| Purpose | Suggested entity |
| --- | --- |
| Solar production total | `sensor.gosungrow_virtual_YOUR_ESS_PS_KEY_p13134` |
| Grid import total | `sensor.gosungrow_virtual_YOUR_ESS_PS_KEY_p13148` |
| Return to grid total | `sensor.gosungrow_virtual_YOUR_ESS_PS_KEY_p13175` |
| Battery charge total | `sensor.gosungrow_virtual_YOUR_ESS_PS_KEY_p13034` |
| Battery discharge total | `sensor.gosungrow_virtual_YOUR_ESS_PS_KEY_p13035` |

Useful supporting sensors outside the Energy config:

| Purpose | Suggested entity |
| --- | --- |
| Battery SOC | `sensor.gosungrow_virtual_YOUR_ESS_PS_KEY_p13141` |
| Daily PV yield | `sensor.gosungrow_virtual_YOUR_ESS_PS_KEY_p13112` |
| Daily PV used by loads | `sensor.gosungrow_virtual_YOUR_ESS_PS_KEY_p13116` |
| Daily PV to battery | `sensor.gosungrow_virtual_YOUR_ESS_PS_KEY_p13174` |
| Daily PV feed-in | `sensor.gosungrow_virtual_YOUR_ESS_PS_KEY_p13173` |
| Daily grid import | `sensor.gosungrow_virtual_YOUR_ESS_PS_KEY_p13147` |
| Current load power | `sensor.gosungrow_virtual_YOUR_ESS_PS_KEY_p13119` |
| Current purchased power | `sensor.gosungrow_virtual_YOUR_ESS_PS_KEY_p13149` |
| Current export power | `sensor.gosungrow_virtual_YOUR_ESS_PS_KEY_p13121` |
| Current battery charge power | `sensor.gosungrow_virtual_YOUR_ESS_PS_KEY_p13126` |

Replace `YOUR_ESS_PS_KEY` with your actual Sungrow ESS device key.

Example from Andreas Roth's system:

- `5072099_14_1_1`

That gives entity IDs such as:

- `sensor.gosungrow_virtual_5072099_14_1_1_p13141`
- `sensor.gosungrow_virtual_5072099_14_1_1_p13173`
- `sensor.gosungrow_virtual_5072099_14_1_1_p13174`

## Alternative: Custom Sungrow Flow Card

If you want something closer to the old README screenshots, use the custom flow card.

### Install The Custom Flow Card

1. Copy the `docs/SungrowEnergy2*.png` files into Home Assistant `/config/www/`.
2. Add the YAML from `docs/examples/home-assistant-sungrow-flow.yaml` as a manual dashboard view or card.
3. Replace `YOUR_ESS_PS_KEY` with your actual ESS device key.

This custom flow card uses the calculated entities generated by GoSungrow, such as:

- `sensor.gosungrow_virtual_YOUR_ESS_PS_KEY_pv_to_load_power`
- `sensor.gosungrow_virtual_YOUR_ESS_PS_KEY_pv_to_grid_power`
- `sensor.gosungrow_virtual_YOUR_ESS_PS_KEY_battery_to_load_power`
- `sensor.gosungrow_virtual_YOUR_ESS_PS_KEY_grid_to_load_power`
- `binary_sensor.gosungrow_virtual_YOUR_ESS_PS_KEY_pv_to_load_power_active`

## Which Path To Choose

Use the official energy cards if:

- you want the cleanest long-term Home Assistant setup
- you want built-in live and historical flow cards
- you want less custom maintenance

Use the custom Sungrow flow card if:

- you specifically want the old Sungrow-style visual flow image
- you want direct control over exactly how the arrows and labels look
